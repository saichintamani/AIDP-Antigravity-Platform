# Failure Registry

## The Philosophy of Failure
*Every successful case will tempt the project to claim more than the evidence supports. Every failed case will tempt the project to explain the failure away. We must resist both.*

> [!WARNING]  
> **[PRELIMINARY / UNVERIFIED (AWAITING LIVE DATA)]**  
> The failures logged below are qualitative observations from early alpha testing. The formal Phase R3 Failure Registry requires the completion of the live N=10 benchmark run to accurately catalog the engine's current error bounds.

This document is the ultimate repository of AI mistakes, false positives, and conceptual dead-ends. It is published publicly to maintain institutional integrity.

## Logged Failures (Historical Replay)

### 1. The String Theory Bias (Early Alpha)
- **Incident:** During initial development, the engine consistently prioritized string theory solutions for the Gravitational Waves case, ignoring the physical interferometry constraints.
- **Root Cause:** LLM training data overwhelmingly over-represents theoretical string physics over experimental engineering.
- **Resolution:** Implementation of the strict Adversarial Peer Review agent to violently enforce mathematical and physical constraints against theoretical drift.

### 2. Constraint Hallucination (H. Pylori)
- **Incident:** The engine generated an experiment assuming acid-suppression was a prerequisite, despite explicit constraints that the pathogen thrived in highly acidic environments.
- **Root Cause:** Latent conceptual blending in the context window.
- **Resolution:** Prompts updated to explicitly inject ontological relationships ("Pathogen -> thrives_in -> Acid").

### 3. Human-AI Disagreement (Pending)
- **Incident:** Reserved for instances where the engine generates a valid hypothesis that the Human Pilot cohort unanimously rejects.
- **Status:** Awaiting N=10 Human Pilot completion.

## Conclusion
A discovery engine is only as valuable as its known failure boundaries. We do not hide these failures; we build atop them.
