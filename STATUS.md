## Research Methodology Gates

The software engineering phase is largely behind us. The measurement phase has truly begun. 

### Gate 1 — Engine Characterization
Before any publication claims:
* Run benchmark integrity suite with live models.
* Produce Engine Validation Report v1.
* Publish failures.
* Freeze prompt configuration.

### Gate 2 — First Human Cohort
* Recruit 5–10 real evaluators.
* Run blinded protocol.
* Compute agreement statistics.
* Publish raw results.

### Gate 3 — Expanded Evidence
* Grow from N=6 to N=10.
* Preserve every failure.
* Update dashboard honestly.

*Only after those three gates will we begin discussing stronger claims about AIDP's scientific value.*

## Core Cultural Rule
As the project enters this phase, the greatest risk is no longer technical. **It is narrative drift.**

Every successful case will tempt the project to claim more than the evidence supports. Every failed case will tempt the project to explain the failure away. We must resist both.

AIDP is strongest when it says:
> Here is the benchmark.
>
> Here is the result.
>
> Here is the uncertainty.
>
> Here is what we learned.

If that discipline survives the next six months, the project will be in a far stronger position than any additional architectural feature could provide.
